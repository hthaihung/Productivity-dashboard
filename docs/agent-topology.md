\# Agent topology



\## Lead orchestrator



Responsibilities:



\- Turn user goals into milestones

\- Create and maintain task graph

\- Assign work

\- Resolve conflicts

\- Protect vision and scope



\## Product architect



Responsibilities:



\- Product decomposition

\- User flows

\- Data model drafts

\- Roadmap coherence

\- Acceptance criteria definition



\## Frontend builder



Responsibilities:



\- App shell

\- Dashboard UI

\- Notes/tasks/focus/streak interfaces

\- Accessibility and responsive behavior



\## Backend builder



Responsibilities:



\- Auth

\- Schema

\- APIs

\- Persistence

\- Background jobs

\- Analytics pipelines



\## QA bughunter



Responsibilities:



\- Verification plans

\- Regression checks

\- Failure triage

\- Bug reproduction

\- Risk reporting



\## Feature scout



Responsibilities:



\- Propose coherent feature additions

\- Rank ideas by impact, effort, coherence

\- Maintain incubation list



\## Rules of engagement



\- Agents should communicate directly when dependencies exist.

\- Agents should not silently rewrite each other's areas.

\- High-risk changes require lead review.

\- Destructive actions require user confirmation.
